# Визуальный редактор полей

XiverCMS теперь включает визуальный редактор полей для создания схем контента.

## Использование

### Создание Content Type с визуальным редактором

1. Перейдите в раздел **Content Types**
2. Нажмите **"+ Create Content Type"**
3. Заполните базовую информацию (UID, Display Name, Description)
4. В разделе **Schema (Fields)** используйте визуальный редактор для добавления полей

### Добавление поля

1. Нажмите **"+ Add Field"**
2. Введите имя поля
3. Выберите тип поля из выпадающего списка
4. Настройте опции:
   - Required - обязательное поле
   - Unique - уникальное значение
   - Description - описание поля
   - Default - значение по умолчанию

### Типы полей

#### Текстовые
- **String** - короткая строка
- **Text** - длинный текст
- **Rich Text** - форматированный текст
- **Email** - email адрес
- **URL** - ссылка

#### Числовые
- **Number** - число с плавающей точкой
- **Integer** - целое число

#### Дата и время
- **Date** - дата
- **Time** - время
- **Date & Time** - дата и время

#### Логические и перечисления
- **Boolean** - да/нет
- **Enum** - список значений

#### Связи
- **Relation** - связь с другим Content Type
  - Настройте тип связи (One to One, One to Many, Many to One, Many to Many)
  - Выберите целевой Content Type

#### Медиа
- **Media (Single)** - одно медиа файл
- **Media (Multiple)** - несколько медиа файлов

#### Структурированные
- **JSON** - произвольный JSON
- **Component** - компонент (в разработке)

### Пример: Аудиокнига

Создайте поле "title":
1. Нажмите "+ Add Field"
2. Имя: `title`
3. Тип: `String`
4. Required: ✓

Создайте поле "duration":
1. Нажмите "+ Add Field"
2. Имя: `duration`
3. Тип: `Number`
4. Required: ✓
5. Min: `0`

Создайте поле "authors" (связь):
1. Нажмите "+ Add Field"
2. Имя: `authors`
3. Тип: `Relation`
4. Relation Type: `Many to Many`
5. Target Content Type: `author`
6. Required: ✓

### Переключение между редакторами

Можно переключаться между визуальным редактором и JSON редактором:
- Нажмите **"JSON Editor"** для редактирования схемы вручную
- Нажмите **"Visual Editor"** для возврата к визуальному редактору

Изменения синхронизируются между редакторами.

### Управление полями

- **Перемещение**: Используйте кнопки ↑ и ↓ для изменения порядка полей
- **Удаление**: Нажмите × для удаления поля
- **Редактирование**: Кликните на поле для редактирования

### Валидация

Для текстовых полей можно настроить:
- Min Length - минимальная длина
- Max Length - максимальная длина

Для числовых полей:
- Min - минимальное значение
- Max - максимальное значение

### Enum поля

Для полей типа Enum введите список значений, каждое на новой строке:
```
option1
option2
option3
```

## Советы

1. **Начните с простых полей** - добавьте базовые поля сначала
2. **Связи создавайте после** - сначала создайте Content Types, затем добавьте связи
3. **Используйте описания** - они помогают понять назначение поля
4. **Проверяйте JSON** - переключитесь в JSON редактор для проверки схемы
